Created: 2026-04-15

## 현재 상태

### 완성된 것
- AST 기반 범용 코드 트레이싱 엔진 (재귀 + 비재귀 모두 지원)
- Web Worker 샌드박스 실행
- 프리셋 시각화 페이지 (`/visualize/[algorithm]`)
- 커스텀 코드 시각화 페이지 (`/visualize/custom`)
- 코드 하이라이팅 (Shiki), 변수 추적, 콜스택, 호출 트리
- TypeScript 스타일 인자 표시 (`arr: number[] = [5,3,8,1,2]`)
- 카테고리별 알고리즘 관리 (코드 파일 분리)
- SEO, PostHog 분석, Vercel Analytics
- Vercel 배포

### 현재 프리셋
| 카테고리 | 알고리즘 | 난이도 |
|----------|----------|--------|
| 재귀 | 순열 (Permutations) | easy |
| 재귀 | 조합 (Combinations) | easy |
| 재귀 | 부분집합 (Subsets) | easy |
| 정렬 | 버블 정렬 (Bubble Sort) | easy |

## Phase 1 — 알고리즘 확장 (무료)

기본 콘텐츠를 충분히 채워서 사이트 가치를 높인다.

### 정렬 (Sorting)
- [ ] 선택 정렬 (Selection Sort) — easy
- [ ] 삽입 정렬 (Insertion Sort) — easy
- [ ] 퀵 정렬 (Quick Sort) — medium
- [ ] 병합 정렬 (Merge Sort) — medium

### 재귀 · 백트래킹 (Recursion & Backtracking)
- [ ] 피보나치 (Fibonacci) — easy
- [ ] 하노이의 탑 (Tower of Hanoi) — easy
- [ ] N-Queen — medium
- [ ] 미로 탐색 (Maze Solver) — medium

### 탐색 (Search)
- [ ] 이진 탐색 (Binary Search) — easy
- [ ] DFS (그래프) — medium
- [ ] BFS (그래프) — medium

### 동적 프로그래밍 (DP)
- [ ] 피보나치 메모이제이션 — easy
- [ ] 배낭 문제 (Knapsack) — medium
- [ ] 최장 공통 부분 수열 (LCS) — hard

### 이 Phase에서 함께 할 것
- [ ] 카테고리 추가: `search`, `graph`, `dp` 섹션을 홈에 표시
- [ ] 난이도 필터 or 정렬 기능

## Phase 3 — 유료 기능 준비

무료 기능으로 트래픽을 확보한 뒤, 유료 기능을 추가한다.

### 유료화 후보 기능
| 기능 | 설명 | 난이도 |
|------|------|--------|
| **AI 코드 분석** | 코드를 넣으면 시간/공간 복잡도, 개선점을 AI가 분석 | 높음 |
| **커스텀 시각화 저장** | 로그인 후 자기 코드/결과를 저장하고 관리 | 중간 |
| **비교 모드** | 두 알고리즘을 나란히 실행해서 성능 비교 | 중간 |
| **임베드** | 블로그/노션에 시각화를 iframe으로 삽입 | 낮음 |
| **고급 알고리즘 팩** | 세그먼트 트리, 네트워크 플로우 등 고급 프리셋 | 낮음 |
| **교육자 모드** | 학생용 링크 생성, 진도 추적 | 높음 |

### 인프라
- [ ] 인증 시스템 (NextAuth / Supabase Auth)
- [ ] 결제 연동 (Stripe / 토스페이먼츠)
- [ ] 사용자 데이터 저장 (Supabase / PlanetScale)
- [ ] 요금제 설계 (무료 / Pro / 교육자)

### 무료 vs 유료 경계선 (안)
| 무료 | 유료 |
|------|------|
| 모든 프리셋 알고리즘 | AI 코드 분석 |
| 커스텀 코드 실행 (제한 없음) | 코드 저장 & 관리 |
| 기본 시각화 | 비교 모드 |
| | 임베드 기능 |
| | 고급 알고리즘 팩 |

## 우선순위 원칙

1. **콘텐츠 먼저** — 알고리즘이 많아야 사람이 온다
2. **무료로 충분히 유용하게** — 유료는 편의/고급 기능이지, 기본 기능을 잠그지 않는다
3. **유저 데이터 기반** — PostHog 이벤트를 보고 어떤 알고리즘이 인기인지, 커스텀을 얼마나 쓰는지 확인 후 유료 기능 결정
