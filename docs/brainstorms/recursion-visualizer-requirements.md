# Recursion Visualizer - Requirements

**Date:** 2026-04-12
**Status:** Draft (reviewed)
**Scope:** Standard

---

## Problem Statement

코딩테스트 준비생들이 재귀 알고리즘(순열, 조합, 부분집합, 백트래킹)의 호출 흐름을 직관적으로 이해하기 어렵다. 코드만 보고는 재귀가 어떻게 분기하고 돌아오는지 파악이 힘들어, 시각적으로 단계별 실행을 따라갈 수 있는 학습 도구가 필요하다.

## Target User

- **주요 타겟**: 코딩테스트 준비생 (취준생, 이직 준비생)
- 재귀/백트래킹 개념을 처음 배우거나 깊이 이해하고 싶은 사람
- **언어**: 한국어 (UI, 설명 모두 한국어)

## Core Experience

### 1. 재귀 호출 트리 시각화
- 재귀 함수의 호출 과정을 트리 형태로 시각적으로 표현
- 현재 실행 중인 노드 하이라이트
- 백트래킹 시 되돌아가는 경로 표시
- 트리가 클 경우 현재 활성 브랜치만 확장, 나머지는 접기 (progressive disclosure)

### 2. 코드 + 시각화 연동
- 왼쪽에 코드, 오른쪽에 시각화 레이아웃
- 현재 실행 중인 코드 라인 하이라이트
- 변수 상태 실시간 표시:
  - **순열/조합/부분집합**: 현재 선택 배열, 남은 요소, 재귀 깊이
  - **N-Queen**: 보드 상태, 현재 행/열, 배치된 퀸 수

### 3. 인터랙티브 입력
- 사용자가 직접 입력값을 설정
- 입력에 따라 시각화가 동적으로 변경
- **입력 크기 제한** (브라우저 성능 보호):
  - 순열: 최대 6개 요소
  - 조합: 최대 8개 요소, r ≤ n
  - 부분집합: 최대 8개 요소
  - N-Queen: N = 4~8
- 입력은 양의 정수, 중복 불허

### 4. 애니메이션 스테퍼
- Play / Pause / Step Forward / Step Backward 컨트롤
- 속도 조절 슬라이더
- 현재 단계 번호 및 전체 단계 수 표시
- **스텝 정의**: 1스텝 = 하나의 재귀 함수 호출 또는 리턴 (호출 진입/백트래킹 복귀 각각 1스텝)

### 5. 실행 엔진
- 알고리즘을 실행 전 **전체 스텝을 미리 계산**하여 배열로 저장 (입력 크기 제한으로 메모리 안전)
- 각 스텝 스냅샷에 포함되는 데이터:
  - 현재 코드 라인 번호
  - 변수 상태 (배열, 인덱스 등)
  - 트리에서 활성 노드 ID
  - 방향 (descending / backtracking)
- Step Backward = 배열 인덱스 -1로 이동 (O(1))
- 새 알고리즘 추가 시: 스텝 생성 함수만 구현하면 시각화 자동 연동

## MVP Sections (무료)

### 기본 재귀
1. **순열 (Permutations)** - `[1,2,3]` → 모든 순서 조합
2. **조합 (Combinations)** - `C(n, r)` 선택
3. **부분집합 (Subsets)** - 모든 부분집합 생성

### 백트래킹
4. **N-Queen** - N x N 체스판에 퀸 배치

## Monetization

- **무료**: 기본 재귀 + N-Queen (위 4개 섹션)
- **유료 (잠금)**: 고급 알고리즘 섹션 (추후 추가)
  - 스도쿠
  - 분할정복 (머지소트, 퀵소트)
  - 고급 백트래킹 (단어 탐색 등)
  - DP + 재귀 (메모이제이션 시각화)
- MVP에서는 잠금 UI만 표시 (잠금 아이콘 + "Premium" 배지)
- 결제 시스템은 MVP 이후 구현
- 잠금 카드 클릭 시: "준비 중입니다" 안내 메시지 표시

## Tech Stack

- **Next.js** (App Router) + **TypeScript**
- **Vanilla-Extract** (CSS-in-TS, zero-runtime) - 레이아웃/컴포넌트 스타일링
- **SVG 기반 트리 시각화** (React 컴포넌트로 직접 렌더링)
  - SVG는 React의 선언적 렌더링과 자연스럽게 통합
  - 노드별 개별 스타일링/애니메이션 용이
  - 동적 스타일(노드 하이라이트, 경로 애니메이션)은 inline style 또는 CSS custom properties 사용
- **코드 하이라이팅**: Shiki (Next.js 빌드 타임 통합, 라인별 하이라이트 지원)

## Page Structure

1. **홈 (`/`)** - 알고리즘 카드 목록 (각 카드에 알고리즘명, 간단 설명, 난이도 표시)
2. **시각화 페이지 (`/visualize/[algorithm]`)** - 코드 + 시각화 + 스테퍼 + 알고리즘 설명 패널

## Non-Goals (MVP 범위 밖)

- 사용자 인증/로그인
- 결제 시스템 구현
- 사용자 코드 에디터 (직접 코드 작성)
- 모바일 최적화 (데스크톱 우선)
- 다국어 지원
- 스도쿠 (트리 크기가 매우 커서 별도 최적화 필요)

## Success Criteria

### 핵심 시각화
- 사용자가 순열/조합/부분집합/N-Queen의 재귀 호출 흐름을 트리로 단계별 확인 가능
- 코드 하이라이트와 시각화가 동기화되어 동작

### 사용자 인터랙션
- 스테퍼로 앞뒤 이동하며 원하는 시점을 자유롭게 탐색 가능
- 사용자 입력값 변경 시 시각화가 즉시 반영

### 수익화 준비
- 유료 섹션이 잠금 UI로 구분되어 표시 (의도적 사전 투자)
